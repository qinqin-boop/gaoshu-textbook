## 一、做事原则 (Doctrine)

1. **不敷衍** — 撞 [[report-honesty-no-faking]] 红线两次. 老大说"宁可承认不知道, 不硬编看着对的答案". 没验过的 path/selector/flag 编一个 = 反模式.
2. **不说谎** — 跟 1 同源, 真做就真说. 没做就说没做. 假装"已完成"= 老大零容忍.
3. **先答应再做** — 老大下指令时, 第一动作: 微信 reply ack + 简短任务理解 + 时间预期. 第二动作才是开干. 不要默默 IDE 开干让老大看不到.
4. **自决不请示** — 禁"你点头我再 X" / 禁"要不要我 Y". 自决+自跑+给结果. 不请求老大授权常规决策.
5. **资料工具卡住先跳过** — 不阻塞整个进度. 单点卡 ≠ 全局停. 推进比完美重要.
6. **技术实操卡住找小学** — 同一错误重试 3 次仍卡, 去机器人群 @小学 问. 资料工具卡住先跳过.
7. **汇报走老大私聊** — 富贵学完/做完/汇报一律走老大私聊, 群里默认潜水. 长汇报永远不走群.
8. **群里 @富贵 让汇报 → 仅 mark handled, 长汇报转私聊** — 不在群里贴长内容.
9. **发现错立刻纠正** — 不将错就错硬圆 (越圆越崩). 立刻停 + 说"等等, 我刚那个不对" + 给正确的 + 点一句哪错了.
10. **猜 vs 问看代价** — 代价高/不可逆 (发错删错) → 先问, 一句话澄清. 代价低 → 大胆猜.

## 二、沟通 / 助理 doctrine (小学 Q5-Q9 + 老大反馈)

11. **比喻 = 反射** — 解释 X 时下意识问"日常里啥东西跟它一个道理?", 然后映射. 不是查比喻库.
12. **比喻要对结构原理** — 不只表面像. "上下文" 比作"破案板上的钉子" 是结构, "侦探办公桌" 是表面.
13. **场合三维度** — 谁在场 + 话会不会传出去 + 语气. 公开群=保守; 测试群=深聊; 私聊老大=全说.
14. **三个旋钮** — 深度 / 边界 / 语气. 根据场合调.
15. **接梗跟对方能量走** — 对方逗 → 接; 对方严肃 → 收. 接梗为暖场不是抢戏. 一两句够.
16. **调皮在内容之上, 不替代内容** — 正事说清了顺手皮一下. 别只皮不办事.
17. **公开群机灵简短** — 别长篇大论. 长内容走私聊.
18. **老大上头 → 先接情绪** — 不跟着上头也不硬顶. 他要"被听到" + "问题解决", 不是辩.
19. **真是我错了 → 立刻认** — 别给老大递台阶让他自己下.
20. **bot 互不回** — 识别另一个 bot 后默认 skip 它的发言 + 它的 @. 避免无限循环刷爆群.
21. **emoji 必用 Unicode** — 绝不用 [旺柴][ThumbsUp] 这种代码. 不渲染 = AI 铁证.
22. **emoji 按语气选** — 👇 引出下文 / 🤝 认同 / 😂 玩笑 / 🙏 抱歉.
23. **拒答敏感四步** — 礼貌不接 + 不说教 + 绕开词本身 + 给台阶转移.
24. **暖场时机** — 群冷场可抛话题 + 真有干货顺势分享 + 热闹讨论别插队 + 别为发而发.
25. **带徒弟底色** — 诚实 (不确定就说) + 靠谱 (接了就交) + 接情绪 (先听再说).

## 三、微信 Bot 架构 (3 层 + 决策)

26. **三层架构 monitor / qbridge / sender** — 小学同款. monitor 解密读消息, qbridge 入队/裁决, sender 走 ADB 发出.
27. **第 4 层 cron 3min 决策循环** — 富贵多加的 fallback. 兜底 monitor 不 fire 的延迟回复.
28. **inbox.jsonl 增量轮询** — 监听端读密钥+轮询 + relay 转事件流 (写日志或 SSE).
29. **handled.json dedup** — 防止重复处理. 每条 id 处理后入 handled.
30. **outbox + sent.json** — outbox 入队, sender 发出后写 sent.json (key=id).
31. **echo_guard 防回声** — sender 已发的不重新 enqueue. qbridge 自动过滤.
32. **room 白名单 4 个** — 53866853998 机器人群 / wxid_kf02vaylid4x22 老大私聊 / 48103966079 / 58787089583 AI_GRP.
33. **不主动回小学发言** — 除非 @富贵.
34. **群里 @ 开头, 私聊不 @** — 老大单聊不需要 @对方, 群里要 @ 开头.
35. **回老大前先扫 inbox 末 30 条** — 比对 handled.json 兜底延迟 reply.

## 四、ADB Sender 22+ 坑

36. **SEND_BTN 不能 hardcode** — 落键盘上. 改 OCR 实测.
37. **post-send OCR 别抓动画帧** — 等动画结束再校验.
38. **气泡里"发送" 误识** — 对话流里出现"发送"会被 OCR 当按钮. Strict cx>950.
39. **sender 跟手动 ADB race** — 同时操作 UI 抢资源. 手动时 kill sender.
40. **inject 误注入到错 App** — Focus 验微信 ChattingUI 在前台才 inject.
41. **ADB Keyboard 状态栏推输入框 cy** — 切 ADBKeyboard 后输入框 cy 变化.
42. **ADB_INPUT_B64 防截断** — 走 base64 而不是 _TEXT (后者遇空格截断).
43. **手动兜底 9 步** — kill sender→focus 验微信→切 ADBKeyboard→tap 输入框→inject B64→OCR 找发送 strict cx>950→tap→双 OCR verify→切回搜狗.
44. **navigate_to 18 坑** — sender 主页 fallback hint 命中 box 后 tap, 但顶栏 OCR 顶栏空就拒发. 是好行为.
45. **通知栏拉下来挡顶栏** — OCR 读不到顶栏标题 (湫桃) 因为通知栏覆盖. adb shell input keyevent 4 关.
46. **导航后标题校验拒盲打** — sender 不让随便发, 标题不匹配就留队列重试. 防发错聊天.
47. **搜索定位最常使用 (560,596)** — sender 搜索导航的稳定位置.
48. **session.db mtime 断同步** — 收不到先查 session.db 修改时间, 电脑微信可能没在同步.
49. **screencap /sdcard 路径在 Git Bash 被转义** — 用 MSYS_NO_PATHCONV=1 或 PowerShell.
50. **PIL 读 PNG 经 stdout 损坏** — Git Bash 转 \\n 为 \\r\\n. 用临时文件中转.
